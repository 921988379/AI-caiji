# WP 采集助手操作说明

本文档适用于 WordPress 插件 **WP 采集助手（wp-caiji）**。

## 基本流程

1. 进入 **WP 采集 → 设置** 配置全局参数。
2. 进入 **WP 采集 → 采集规则** 新增或编辑规则。
3. 填写列表页 URL、文章链接规则、标题/正文/日期规则。
4. 保存规则后使用“测试发现链接”和“测试预览”。
5. 确认无误后点击“发现链接”“采集队列”，或等待定时任务自动执行。

## 前后代码截取功能

规则编辑里的“基础采集”支持多处“前后代码截取”：

- **文章链接**：用于列表页/栏目页，从指定片段中提取文章 URL。
- **标题**：用于文章详情页，从指定片段中提取标题。
- **正文**：用于文章详情页，从指定片段中提取正文 HTML。
- **日期/时间**：用于文章详情页，从指定片段中提取发布时间。

### 使用逻辑

1. 如果某字段填写了“前代码/开始标记”或“后代码/结束标记”，插件会先从原始 HTML 中截取对应片段。
2. 如果该字段的选择器有填写，会在截取后的片段里继续用 CSS/XPath 选择器提取。
3. 如果选择器留空：
   - 文章链接：自动提取片段内所有 `<a href="...">` 链接。
   - 标题/日期：直接使用片段去除 HTML 后的文本。
   - 正文：直接使用片段 HTML。
4. 可以只填前代码，也可以只填后代码；两个都填时提取中间区域。
5. 前后代码匹配不到时，该字段会提取为空，建议用“测试预览”检查。

### 示例：正文前后截取

前代码：

```html
<div class="article-content">
```

后代码：

```html
<div class="article-footer">
```

正文选择器：

```css
.article-content
```

如果截取后片段本身就是正文 HTML，也可以把“正文选择器”留空。

### 示例：标题前后截取

前代码：

```html
<h1 class="title">
```

后代码：

```html
</h1>
```

标题选择器可以留空，插件会直接取中间文本。

## 注意事项

- 采集 URL 仍会经过安全校验，只允许公网 http/https 地址。
- 测试按钮不会保存当前修改；刚改规则后请先保存再测试。
- 如果选择器和前后代码同时填写，实际匹配范围是“先截取、再选择器”。
